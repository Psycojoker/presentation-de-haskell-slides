Les fonctions

  • elles sont pures
  • pas possible d'acceder un contexte globale et de le modifier
  • pas de "variable"/modification de variable
  • ne prennent qu'une seule expression
  • est obligatoirement retourne la valeur de l'expression

  • impossible d'y faire des effets de bords (side-effect)
  • rend haskell trs solide (car empche plein de sources de bugs)
  • rend haskell trs difficile

17/49

Encore plus de pattern matching

Rappel sur les listes:

1 [1, 2, 3] == 1:2:3:[]

On peut faire du pattern matching dessus:

1 head (x:xs) = x -- les () sont l pour la priorit d'valuation
2 
3 tail (x:xs) = xs
4 
5 len [] = 0
6 len (x:xs) = 1 + len xs

Exemple, "len [1,2,3]" donnera:

len [1,2,3]
match len (x:xs)
len (1:[2,3]) = 1 + len [2,3]
-> len (2:[3]) = 1 + len [3]
-> len (3:[]) = 1 + len []
-> len [] = 0

Donc:
len [1,2,3] = 1 + 1 + 1 + 0

Quicksort

Pour la blague:

quicksort [] = []
quicksort (x:xs) = (quicksort [a | a <- xs, a <= x]) ++ [x] ++\

1 (f o g)(x) == f(g(x))

1 a . b = (\x -> a $ b $ x)
2

data Shape = Circle Float Float Float | Rectangle Float Float Float Float deriving (show)

Circle :: Float -> Float -> Float -> Shape

Rectangle :: Float -> Float -> Float -> Float -> Shape

surface :: Shape -> Float
surface (Circle _ _ r) = pi * r ^ 2
surface (Rectangle x1 y1 x2 y2) = (abs $ x2 - x1) * (abs $ y2 - y1)

314.15927
ghci> surface $ Rectangle 0 0 100 100

Maybe

Certaines fonctions sont unsafe.

Exemple:

head [] -- Boom!

Problem: pas de None car le None SMALE !

Solution: le maybe.

data Maybe a = Just a | Nothing

safeHead :: [a] -> Maybe a
safeHead [] = Nothing
safeHead (x:_) = Just x

haveAnHead :: Maybe a -> String
haveAnHead Just _ = "Yeah, have an head!"
haveAnHead Nothing = "No head, no chocolate"
